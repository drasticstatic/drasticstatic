## What I build in the open

Nearly everything I code eventually surfaces as open, public work — tools that I once needed to exist but couldn't find elsewhere and sharing with others 🙂

**Public repositories include:**

- **[Trading Assistant](https://drasticstatic.github.io/trading-assistant-public-preview/)** — AI-powered futures trading workflow using Claude Code, MCP servers, Pine Script, and TradingView integrations. Session reviews, pre-market plans, pattern tracking, and a skills library — all open.
- **[Ethereal Offering](https://drasticstatic.github.io/gratitude-token-project_docs/)** — A gratitude-based DAO with soulbound DIDs, ZK governance, and ritual intelligence for sacred communities and recovery fellowships.
- **[De Vine News (PIR)](https://github.com/drasticstatic/pir-devine-news)** — A committee portal and newsletter platform for Psychedelics In Recovery®, a 12-step recovery fellowship.
- **[Portfolio & Resume](https://drasticstatic.github.io/resume/)** — Open-source portfolio site showcasing the full range of work.
- **Public surfaces** — HTML templates, code snippets, and workflow tooling built so others can learn from and build on them.

- **Infrastructure** — GitHub Actions minutes, hosting, domain costs, MCP server uptime

- **Time** — More hours spent building and documenting public work instead of purely private client work
- **Recovery tech** — Keeping PIR De Vine News and future recovery community tools free and open
